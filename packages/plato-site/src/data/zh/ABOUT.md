# 关于柏拉图

> *他不仅是最善良的人，也是最智慧的人。*

柏拉图（Plato）是一个伪装成解谜游戏的证明助手。它以那位坚信真知源于推理而非被告知的哲学家命名。

我建造它，是因为想要一个逻辑能像玩耍一样的地方。大多数证明助手令人望而生畏：它们说着晦涩的语法，要求先啃完几百页文档，默认你已经有博士学位。柏拉图从另一边出发：你输入 `(assume A)`，看到一行新证明跳出来，心里一动，再打几行——不知不觉间你已经在用蕴含式思考了。

它不是为了让你*了解*逻辑——而是让你*做*逻辑。最好的学习方法永远是亲自动手，犯错，然后慢慢培养出对证明的直觉。

## 如何使用

你会看到一个待证明的命题。在 S-表达式 REPL 中输入命令——`(assume A)`、`(->-elim 2 1)`、`(and-intro 1 3)`——引擎会根据自然演绎规则逐行检查。引导卡片陪你走完最初的证明，提示在你卡住时出现。每解出一道题都会解锁新的战术，一步一步筑起你的工具箱。

目前有 **45 道题目**，从身份律（三步、两分钟）到皮尔士律和排中律，从关于苏格拉底的谓词逻辑到德摩根定律。你也可以从 URL 或 JSON 文件**加载自定义题目**——引擎不在乎目标是从哪里来的。

## 为什么是自然演绎？

因为自然演绎模仿的是人类真实的推理方式。

大多数逻辑课上，你学的是真值表。一行一行填，一列一列查。它确实有效，但感觉不像*思考*。自然演绎不同：你假设、推导、消去假设。每一步都对应一个真实的推理动作——「如果我知道 *p*，那我就知道 *q*。」你不是在计算真值，你是在构建论据。

## 关于难度

自然演绎确实很难。构建证明没有固定算法——你必须培养手感。看起来简单的题目可能极具挑战性。卡住了很正常，放过它，明天再来看，往往会豁然开朗。这不是一个可以速通的新手教程，而是一种需要日积月累的能力。

## 技术

- **引擎**：Rust，通过 `wasm-pack` 编译为 WebAssembly。整个逻辑内核——公式、上下文、断言、推理规则、S-表达式解析器——约 3000 行 Rust。
- **前端**：Vue 3 + TypeScript + Vite。单页应用，没有路由器——仅靠一个判别联合类型和页面过渡动画驱动。
- **渲染**：KaTeX 负责数学公式，JetBrains Mono 负责其余一切。
- **国际化**：完整的中英文支持，包括为每个技术术语提供的双语术语表。
